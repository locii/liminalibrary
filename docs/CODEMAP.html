Introduction

Electron + React desktop app for breathwork facilitators: scan local audio, match against Music for Breathwork (MFB), integrate with Limina Studio (.limina sessions).

Rule: Renderer never calls Node — only window.electronAPI from src/preload/index.ts.

Audio server

startAudioServer() in src/main/index.ts — local HTTP on random 127.0.0.1 port for Chromium playback.

Do not break these constraints (confirmed Chromium/ffmpeg bugs):
1. Resample to 48kHz when source sr > 48000 (2× speed bug).
2. Never transcode via WAV pipe — always ffmpeg → FLAC.
3. Direct-stream WAV only if 16-bit PCM int; else transcode.

Playback URL: http://127.0.0.1:{port}/{encodeURIComponent(path)}?sr={sampleRate}

Direct formats: mp3, m4a, mp4, aac, ogg, oga, opus, webm, flac, and safe 16-bit WAV.

  • scoreMatch(entry, track) — title/artist/album token overlap + folder hints
  • Auto-apply threshold: 0.25
  • mfb:rankMatches returns top 8 with score > 0.05

External APIs

Base: https://musicforbreathwork.com/api (HTTPS from main process only)

  • GET /tracks — catalogue cache (auth expands catalogue)
  • GET /tracks/:id
  • POST /auth/login, POST /auth/logout, GET /auth/me
  • GET /user/playlists, GET /playlist/:id
  • POST /library/sync — { track_ids: number[] }
